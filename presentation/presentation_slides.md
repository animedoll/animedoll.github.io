### 二、 Presentation 逐页内容策划

#### **Slide 1: 封面 (Cover)**
* **主标题**：AnimeDoll —— 你的跨次元 AI 专属陪伴
* **副标题**：嵌入式智能二次元公仔挂件产品策划案
* **视觉建议**：一张精美的3D渲染图，展示一个极具二次元风格的精美小公仔挂在一个透明“痛包”上，公仔的眼睛或配饰有微光（暗示智能），背景是赛博朋克或温馨的日常街道。

#### **Slide 2: 目录 (Table of Contents)**
* 列出上述的四大核心模块：需求分析、概念设计、方案展示、总结展望。

#### **Slide 3: 3.1 前期调研 (背景与 5W1H)**
* **设计背景**：“陪伴经济”崛起，Z世代对二次元周边（谷子）的消费从“收集”转向“情感寄托”。同时，AI可穿戴设备（如AI Pin）兴起，但缺乏针对垂直亚文化群体的设计。
* **5W1H 分析模型**：
    * **Who**：15-25岁泛二次元爱好者、谷子（周边）收集控。
    * **What**：一款10cm大小，具备看（微型摄像头）和说（语音合成）能力的实体AI挂件。
    * **When**：日常通勤、逛街、参加漫展（Comic Con）时。
    * **Where**：挂在痛包、单肩包、书包的显眼位置。
    * **Why**：打破次元壁，提供情绪价值，成为社交破冰的神器。
    * **How**：通过蓝牙连接手机App，调用云端大模型实现视听交互。

#### **Slide 4: 3.1 前期调研 (行业调研与 SWOT)**
* **行业调研**：潮玩市场规模破千亿，但传统毛绒/搪胶公仔同质化严重，缺乏交互；AI硬件多为冰冷的数码形态（如项链、胸针），缺乏温度。
* **SWOT 分析表**：

| 维度 | 分析内容 |
| :--- | :--- |
| **S (优势)** | 独占细分市场；实体周边自带情绪价值；隐蔽性高，外观可爱。 |
| **W (劣势)** | 10cm体积限制了电池容量；散热要求高；硬件成本较高。 |
| **O (机会)** | 大模型API成本断崖式下降；二次元“痛包”文化普及。 |
| **T (威胁)** | 知名动漫IP的授权壁垒；公共场合摄像头的隐私争议。 |

#### **Slide 5: 3.2 用户调研 (画像与洞察)**
* **目标画像**：小A，19岁，大学生。背包上挂满了喜欢的二次元角色周边。她希望这些角色能“活”过来，陪她一起看世界。
* **访谈洞察 (引言提取)**：
    * *“每次去漫展，都希望我的推（喜欢的角色）能看到现场有多热闹。”*
    * *“一个人坐地铁挺无聊的，如果包上的挂件能跟我吐槽两句就好了。”*

#### **Slide 6: 3.3 需求分析 & 3.4 机会点总结**
* **痛点转化为需求**：
    1.  **静态陪伴 -> 动态交互**：需要个性化语音和人设匹配。
    2.  **视觉盲区 -> 共同见证**：需要第一视角摄像头记录并识别环境。
* **机会点总结 (核心切入点)**：**“不仅是装饰，更是有记忆的旅行搭子。”** AnimeDoll 的核心不在于“生产力工具”，而在于“情绪陪伴器”。

#### **Slide 7: 4.1 概念转化与产品定义**
* **产品概念**：装在口袋里的“赛博手办”。
* **产品定义**：基于 BLE（低功耗蓝牙）与手机协同工作的边缘智能硬件。通过摄像头捕获画面传至手机，手机端调用多模态大模型解析后，生成符合特定角色性格的台词，再通过本地/云端TTS转化为角色的声音由挂件播放。

#### **Slide 8: 4.2 商业模式画布**
* **价值主张**：提供打破次元壁的实体情感陪伴。
* **客户细分**：二次元圈层用户、潮玩盲盒玩家、孤独经济消费者。
* **收入来源**：
    1.  硬件销售（公仔本体）。
    2.  软件增值（App内的角色性格包、高级音色库订阅）。
    3.  周边配件（公仔的可替换服装、痛包等）。

#### **Slide 9: 4.3 核心功能模块**
* **看 (视觉感知)**：定时抓拍或触发式识别眼前事物（如识别到另一只AnimeDoll、识别到漫展海报）。
* **说 (性格语音)**：内置扬声器，支持不同声线的TTS（傲娇、呆萌、御姐等），可主动搭话。
* **记 (数字记忆)**：在配套App中生成“公仔视角的日记/手帐”，记录今天和主人去了哪里。

#### **Slide 10: 4.4 系统图与技术路线**
* **架构设计 (强调端云结合，解决功耗问题)**：
    * **挂件端 (Device)**：低功耗MCU + 微型摄像头 (如OV系列) + 麦克风阵列 + 微型喇叭 + 小容量锂电池。只负责采集和播放。
    * **手机端 (App)**：作为算力中枢，处理蓝牙数据流，压缩图片，发起网络请求。
    * **云端 (Cloud)**：LLM多模态大模型（图像理解+文本生成） + AI声学模型（声音克隆/合成）。

#### **Slide 11: 4.5 体验流程与故事版 (Storyboard)**
* **场景连环画 / 流程图**：
    1.  **清晨**：小A把 AnimeDoll 挂在包上出门，捏了一下公仔的爪子（唤醒）。公仔发声：“今天也要元气满满哦，主人！”
    2.  **漫展现场**：公仔胸前的隐藏摄像头拍到了一位高质量Coser，公仔主动发声提示：“哇，11点钟方向捕捉到一只野生初音未来！”
    3.  **晚上回家**：小A打开App，发现公仔自动生成了今天的《漫展漫游手帐》，里面有公仔视角的抓拍和配文，小A一键分享到了社交平台。

#### **Slide 12: 5.1 外观与 CMF 设计 [方案展示]**
* **尺寸与重量**：严格控制在 10cm 以内，重量小于 50g。
* **材质 (CMF)**：头部采用亲肤短毛绒（增加治愈感），身体内部骨架采用轻质ABS塑料（包裹电子元件）。
* **巧妙隐藏**：微型摄像头隐藏在公仔的“纽扣”、“墨镜”或“蝴蝶结”中心，避免突兀感和路人的隐私抵触感。

#### **Slide 13: 5.2 内部结构与硬件布局 [方案展示]**
* **爆炸图展示**：
    * **头部**：微型定向扬声器（声音向下传播，减少对路人打扰）与收音麦克风。
    * **胸部/腹部**：柔性 PCB 主板、蓝牙天线与摄像头模组。
    * **背部**：Type-C 充电口（隐藏在衣服下）及 300mAh 电池。

#### **Slide 14: 5.3 交互界面设计 (Companion App) [方案展示]**
* **UI 视觉风格**：二次元、扁平化、手账风。
* **三大核心界面**：
    1.  **灵魂注入室 (配置页)**：设定公仔的称呼、性格Prompt（如：你是一个毒舌但傲娇的魔法少女）、音色选择。
    2.  **时空相册 (记忆页)**：公仔第一视角的照片流及AI生成的内心OS。
    3.  **能量站 (状态页)**：查看电量、蓝牙连接状态、固件升级。

#### **Slide 15: 5.4 渲染图与使用场景呈现 [方案展示]**
* **大图冲击**：高质量的场景渲染图。
    * 图1：AnimeDoll 挂在一个装满吧唧（徽章）的痛包上，完美融入二次元穿搭。
    * 图2：夜间模式，公仔的眼睛或标志性部位发出微弱的呼吸灯光。

#### **Slide 16: 6. 总结与未来展望**
* **阶段规划**：
    * Phase 1：推出无属性原创设计的 AnimeDoll，验证硬件与交互逻辑。
    * Phase 2：开启IP联名计划（如初音未来、原神等官方授权），售卖专属“角色灵魂包”。
* **愿景**：“赋予冰冷的周边以灵魂，让每一份热爱都能得到回应。”
